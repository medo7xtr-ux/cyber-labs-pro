# منصة المعامل الأمنية (CyberLabs Platform)

منصة تعليمية تفاعلية لتعلم أمن المعلومات واختبار الاختراق، مصممة لتوفير بيئة تعليمية عملية للمهتمين بالأمن السيبراني.

## 🚀 الميزات الجديدة والمطورة
- **نظام تقييم آلي:** تقييم فوري لتسليمات المستخدمين (الأعلام والنصوص).
- **بيئة مؤمنة:** إعدادات متقدمة للأمان واستخدام متغيرات البيئة.
- **إدارة متقدمة للمعامل:** تصنيفات متعددة ومستويات صعوبة متنوعة.
- **لوحة تحكم المستخدم:** تتبع التقدم، النقاط، والإنجازات.
- **توثيق API:** دعم كامل لـ OpenAPI/Swagger.

## 🛠 التقنيات المستخدمة
### الواجهة الخلفية (Backend)
- **Django 5.0 & DRF:** لبناء الـ API.
- **JWT Authentication:** لتأمين الوصول.
- **PostgreSQL/SQLite:** لإدارة البيانات.
- **Channels & Redis:** للوظائف الفورية (WebSockets).

### الواجهة الأمامية (Frontend)
- **React.js & Vite:** لسرعة التطوير والأداء.
- **Material-UI (MUI):** لواجهة مستخدم احترافية.
- **Redux Toolkit:** لإدارة الحالة.
- **TanStack Query:** لجلب البيانات بكفاءة.

## ⚙️ كيفية البدء

### متطلبات التشغيل
- Python 3.10+
- Node.js 18+
- Redis (اختياري للوظائف الفورية)

### إعداد الواجهة الخلفية
1. انتقل إلى مجلد الباك-إند: `cd backend`
2. أنشئ بيئة افتراضية: `python -m venv venv`
3. فعل البيئة: `source venv/bin/activate` (أو `venv\Scripts\activate` على ويندوز)
4. ثبت الاعتماديات: `pip install -r requirements.txt`
5. أنشئ ملف `.env` بناءً على النموذج المرفق.
6. قم بإجراء الترحيلات: `python manage.py migrate`
7. شغل الخادم: `python manage.py runserver`

### إعداد الواجهة الأمامية
1. انتقل إلى مجلد الفرونت-إند: `cd frontend`
2. ثبت الحزم: `npm install`
3. شغل خادم التطوير: `npm run dev`

## 🔒 الأمان
تم إعداد المشروع ليكون متوافقاً مع معايير الأمان الأساسية، بما في ذلك حماية CORS، وإدارة المفاتيح السرية عبر متغيرات البيئة، وتعطيل وضع التصحيح في الإنتاج.
